# Code of Conduct

This project is committed to providing a welcoming and respectful environment for everyone.
Adapted from the [Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

## Our standards

Examples of behavior that contributes to a positive environment:

- Being respectful and considerate
- Gracefully accepting constructive feedback
- Focusing on what is best for the project and community
- Assuming good intent and clarifying misunderstandings

Examples of unacceptable behavior:

- Harassment, discrimination, or hate speech
- Personal attacks or sustained hostility
- Publishing private information without permission
- Disruptive behavior that prevents collaboration

## Enforcement

Project maintainers are responsible for clarifying standards and may take appropriate action in response to unacceptable behavior, up to and including a permanent ban.

## Reporting conduct issues

If you experience or witness unacceptable behavior:

1. **Open a GitHub issue** labeled **"conduct"** with minimal details if you feel safe doing so.
2. **Contact a maintainer directly** via the email or contact link on their GitHub profile (visible on the Contributors page of this repo).

Please do not open detailed public reports if the situation involves personal information or sensitive context - use option 2 in that case.

> Note: `SECURITY.md` is for reporting software vulnerabilities only - please do not use
> that channel for conduct concerns.

## Scope

This code of conduct applies within project spaces and in public spaces when an individual is representing the project.
