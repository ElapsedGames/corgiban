## Summary

benchmarkComplete is dispatched even when the known-solution path does not represent an actual
benchmark completion (solved: false, moveCount: 0, elapsedMs: 0). This misleads embedding apps
that treat the event as proof of benchmark completion.

## Expected

Either rename the event semantics for this path, or ensure the payload contract clearly indicates
synthetic or no-benchmark completion so embedding apps can distinguish it.

## Actual

benchmarkComplete fires with a zero/false payload on the known-solution path.
EmbedView.tsx:316, :349.

## Repro

Render embed with a known-solution level and observe the benchmarkComplete event payload.

## Notes

Source: Review 4 #3.

## Fix Plan

- Option A: rename the event for this code path to distinguish it from real benchmark completion
- Option B: add a field to the payload (e.g. synthetic: true) that embedding apps can check
  Document the chosen contract in the embed README.
