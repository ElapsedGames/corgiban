## Summary

disconnectedCallback tears down the React root or state but does not explicitly clear shadow DOM
content or style nodes. Under repeated mount/unmount cycles or exotic host reinitialization
patterns, stale content can accumulate and make behavior timing-dependent.

## Expected

disconnectedCallback should explicitly clear shadow DOM content before the React root is torn
down, ensuring remount paths start from a clean state.

## Actual

Shadow DOM content is not cleared. corgibanEmbed.tsx:82, :95.

## Repro

Mount and unmount the custom element multiple times in rapid succession and inspect shadow DOM.

## Notes

Source: Review 4 #4. Exotic host pattern -- low reproduction likelihood in normal use.

## Fix Plan

Add explicit shadowRoot.innerHTML = '' and detach root state before remount paths in
disconnectedCallback.
