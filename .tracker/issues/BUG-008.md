## Summary

startedAtMs and finishedAtMs are captured with Date.now(). If the system clock moves backward
during a run, the emitted record has finishedAtMs < startedAtMs, which fails strict validation
at benchmarkSchema.ts:267 on export/import.

## Expected

Either the schema allows this edge case, or timestamps are captured in a way that prevents
backward time (e.g. Math.max(startedAtMs, Date.now()) for finishedAtMs).

## Actual

Clock rollback produces an invalid record that fails schema validation on export/import.

## Repro

Mock Date.now() to go backward mid-run and observe schema validation failure.

## Notes

Source: Review 3. benchmarkPort.client.ts:237, :315, :354. benchmarkSchema.ts:267.

## Fix Plan

- Option A: clamp finishedAtMs to be >= startedAtMs
- Option B: document as a known environmental edge case and accept the validation failure
  Decision should be documented.
