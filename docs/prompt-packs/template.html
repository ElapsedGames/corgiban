OUTPUT section inside the Planning (phase-one) panel. This contains the scope lock, coverage map,
and acceptance criteria that define what changed in this PR.

Review every documentation file that may be affected by all uncommitted, unstaged, and untracked changes in the repo:
- docs/Architecture.md - update if any architectural boundaries, module structure, or data flow changed.
- docs/adr/ - create a new ADR if any architectural decision was made (new pattern, trade-off chosen, approach locked in).
- packages/*/README.md - update each package README to reflect new exports, public API, or usage examples added in this PR.
- docs/project-plan.md - mark completed tasks/phases as done if applicable.
- LLM_GUIDE.md - update if new conventions, patterns, or constraints were established.
- CONTRIBUTING.md - update if developer workflow, setup steps, or tooling changed.

For each file:
1. Read it in full.
2. Identify any section that is stale, missing, or contradicts what was built.
3. Update or create as needed.

Do not mark this tab complete until every affected doc has been reviewed and either updated or explicitly confirmed as current.
